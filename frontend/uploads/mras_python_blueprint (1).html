01 Core Philosophy & Stack
1.1 Guiding Principles
Async by Default
Every endpoint and DB call uses async/await. Never block the event loop with synchronous I/O.
Type Everything
All request bodies, responses, and DB models use Pydantic v2 + SQLModel. No untyped dicts crossing module boundaries.
Modular Monolith
Single deployable unit, but cleanly separated modules with explicit interfaces. Easy to split into microservices if needed.
Dependency Diet
Do not add a library if the stdlib or FastAPI built-ins can handle it. Every package in requirements.txt must justify its weight.
Privacy by Design
PHI (protected health information) is AES-256 encrypted at rest. All sensor collection is opt-in only.
Graceful Degradation
Every external dependency (climate API, LLM, wearable) fails safely. Predictions are advisory; no feature blocks core workflows.

3.4 Solving N+1 Query Problem
Always use selectinload() or joinedload() when fetching related data. A loop calling db.get() per patient will kill performance.
# Bad — N queries
patients = await db.execute(select(Patient))
for p in patients:
    p.consultations  # triggers new query each time

# Good — 2 queries total
stmt = (
    select(Patient)
    .options(selectinload(Patient.consultations))
)
result = await db.execute(stmt)
04 Database & ORM (SQLModel)
4.1 Model Definition Pattern
One class serves as both the SQLAlchemy table and Pydantic schema. Separate Create / Read schemas to control what is exposed in API responses.
from sqlmodel import SQLModel, Field
from datetime import datetime

class PatientBase(SQLModel):
    employee_id: str = Field(index=True, unique=True)
    full_name: str
    department: str
    date_of_birth: datetime

class Patient(PatientBase, table=True):  # DB table
    id: int | None = Field(default=None, primary_key=True)
    created_at: datetime = Field(default_factory=datetime.utcnow)

class PatientCreate(PatientBase): ...    # Request body
class PatientRead(PatientBase):           # Response (no secrets)
    id: int
4.2 TimescaleDB for Sensor & Health Data
All time-series data (sensor readings, health check-ins, JRISSI scores) must use TimescaleDB hypertables. Do not store them in standard tables — query performance degrades rapidly at scale.
# In Alembic migration (after CREATE TABLE)
op.execute("""
    SELECT create_hypertable(
        'sensor_reading', 'timestamp',
        if_not_exists => TRUE
    );
""")
4.3 Migration Rules
Every schema change has a migration: alembic revision --autogenerate -m "add_mhrs_column"
Always provide a downgrade() function. Rollback capability is non-negotiable.
Never run SQLModel.metadata.create_all() in production. Use Alembic only.
05 Authentication & RBAC
5.1 Role Definitions
employee doctor pharmacy_staff admin
JRISSI scores are visible to doctor role only. Never expose to admin or any HR-adjacent role. This is a hard privacy requirement.
Sensor data collection requires an explicit opt-in flag on the employee record (sensor_consent: bool). A missing or false flag means no collection.
5.2 JWT Dependency
def require_role(*roles: str):
    async def checker(token = Depends(oauth2_scheme)):
        payload = decode_jwt(token)
        if payload["role"] not in roles:
            raise HTTPException(status_code=403)
        return payload
    return checker

06 ML / AI Layer
6.1 JRISSI Scorer Pattern
# modules/jrissi_scorer.py
MHRS_HIGH_THRESHOLD = 70
MHRS_MODERATE_THRESHOLD = 40

class JRISSIScorer:
    def compute_mhrs(self, employee_id: int, db) -> MHRSResult:
        jri = self._score_jri(employee_id, db) * 0.30
        ssi = self._score_ssi(employee_id, db) * 0.25
        phi = self._score_phi(employee_id, db) * 0.25
        cps = self._score_cps(employee_id, db) * 0.20
        mhrs = round(jri + ssi + phi + cps)

        if   mhrs >= MHRS_HIGH_THRESHOLD:     category = "High"
        elif mhrs >= MHRS_MODERATE_THRESHOLD: category = "Moderate"
        else:                                  category = "Low"

        return MHRSResult(score=mhrs, category=category)
6.2 Claude API — Behaviour Correction Text
import anthropic

async def generate_correction(risk: str, profile: EmployeeProfile) -> str:
    client = anthropic.AsyncAnthropic()
    message = await client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=256,
        system="You are a corporate health advisor. Write one short, "
               "actionable health recommendation (2–3 sentences). "
               "Never diagnose. Always recommend doctor follow-up.",
        messages=[{"role": "user",
                   "content": f"Risk: {risk}. Profile: {profile.summary()}"}]
    )
    raw = message.content[0].text
    return SafetyFilter.validate(raw)  # Always filter before sending
All LLM output passes through SafetyFilter before any delivery. The filter checks for: diagnostic language, medication dosage claims, and links/URLs.
Every generated message displays a disclaimer: "This is an advisory suggestion. Consult your doctor before making health decisions."
6.3 Climate API Fallback Chain
async def get_climate_data(lat: float, lon: float) -> ClimateReading:
    try:
        return await openweathermap_client.fetch(lat, lon)
    except (TimeoutError, APIError):
        logger.warning("OpenWeather failed — trying Tomorrow.io")
        try:
            return await tomorrowio_client.fetch(lat, lon)
        except (TimeoutError, APIError):
            logger.error("Both climate APIs failed — skipping forecast")
            return None  # Caller must handle None gracefully

08 Security & Privacy
8.1 Non-negotiable Rules
AES-256 encryption at rest for all PHI fields (diagnoses, medications, MHRS scores).
TLS 1.3 in transit for all API communication. No HTTP in production.
SQL injection prevention: never use string formatting in queries. Use SQLAlchemy parameterised statements only.
Rate limiting on auth endpoints: max 5 failed login attempts per IP per 15 minutes.
All external links in API responses use rel="noopener noreferrer".
Rotate SECRET_KEY and API keys every 90 days. Store in AWS Secrets Manager, not in .env on the server.
